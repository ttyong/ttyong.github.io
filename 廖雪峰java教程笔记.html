java快速入门

java简介

java最早由sun公司的詹姆斯.高斯林开发编写

最初名为oak，后来发现这是一门已有的计算机语言，所以改名java

java是一门介于解释和编译型的语言

C，C++,代码直接编译成机械码，有cpu执行，但由于不同平台的中央处理器的CPU的指令集不同，因此，需要编译出每一种平台的机器码。

Python,Ruby没有上述问题，直接有解释器加载源码然后运行，但运行效率太低

java是由编译器生成一种独立的字节码，然后针对不同平台开发不同的虚拟机，就可以实现一次编写，到处运行

不同厂商都可以编写自己的虚拟机，为了保证字节码能够正确运行，就需要制定一系列的虚拟机规范

java三个不同的版本：

java se: standard edition

java ee: enterprise edition

java me: micro edition

java ee 在java se的基础上加了一些API和库

java ee和java se 使用的虚拟机完全不同

** JDK和JRE:**

JDK: java development kit

JRE: java runtime environment

JDK包含虚拟机和编译器，调试器，JRE只有虚拟机.因此JDK包含JRE

JSR规范：java specification request 负责统一接口

JCP组织：java comunity process 审核JSP

安装JDK

配置JAVA_HOME:安装java的路径，bin的上一级

配置Path: %JAVA_HOME%\bin

细心的童鞋还可以在JAVA_HOME的bin目录下找到很多可执行文件：

  • java：这个可执行程序其实就是JVM，运行Java程序，就是启动JVM，然后让JVM执行指定的编译后的代码；
  • javac：这是Java的编译器，它用于把Java源码文件（以.java后缀结尾）编译为Java字节码文件（以.class后缀结尾）；
  • jar：用于把一组.class文件打包成一个.jar文件，便于发布；
  • javadoc：用于从Java源码中自动提取注释并生成文档；
  • jdb：Java调试器，用于开发阶段的运行调试。

第一个java程序

一个java程序要有一个主类，文件名和主类相同

编译时要带后缀，运行时不带后缀

使用IDE

IDE: Integrated Development Environment

使用IDE的好处在于按，可以把编写代码、组织项目、编译、运行、调试等放到一个环境中运行，能极大地提高开发效率。

java程序基础

java程序基本结构

类名规范：

  • 首字母大写
  • 字母开头，数字，下划线组合

方法名规范：

  • 首字母小写

public是访问修饰符，表示该类公开。不写public也能正确编译，但这个类无法从命令行执行

每一行语句，分号结尾

注释方法：

  • //
  • /*… */
  • /*… */ *这是一种特殊注释方法，用在类和方法的定义出，用于自动创建文档

*对于eclipse可以用快捷键ctrl+shift+f，快速格式化代码

移位运算符

数值的最高位是一个符号位

》： 右位移

《： 左位移

上面两种不会改变符号位

》>：会改变符号位

对byte和short移位会首先转换为int

右移或左移移位代表在之前的基础上乘或除2的几次方倍

位运算

&：同一才为一

|： 有一则为一

~(非)： 零一互换

^(异或): 不同才为一
